## 前言

你是否担心传统代理方案容易被识别、封锁，或者在使用过程中频繁失效？ **VLESS + Reality** 正是为了解决这一问题而出现的一种更隐蔽、更接近真实网络行为的通信方式。相比以往明显带有"代理特征"的流量，它的目标只有一个： **让你的连接看起来就像普通人在访问一个正常网站。**

传统方案通常依赖标准 TLS 证书或固定协议特征，这些特征一旦被识别，就可能被针对性干扰。而 Reality 的思路完全不同：它不再依赖真实证书，而是**直接模拟一个真实网站的 TLS 握手过程**。从网络视角来看，你只是访问了一个常见的 HTTPS 网站，整个通信过程几乎没有明显异常。

与此同时，VLESS 协议本身被设计得极其简洁——它只负责"确认你是谁"（身份验证），而不负责复杂加密。真正的"伪装"和"安全外衣"交给 Reality 来完成。这种分层设计，让整体连接既轻量高效，又难以被识别。

简单来说，这套方案的核心逻辑是：

> **用 VLESS 做身份识别，用 Reality 把流量伪装成正常 HTTPS 通信**

当连接建立时，客户端会伪装成访问某个真实网站，完成一次"看似正常"的 TLS 握手；只有在握手中通过特定验证的客户端，服务器才会切换到真实的数据通道。对外，它始终像普通流量；对内，它才是真正的代理连接。

正因为这种"以假乱真"的设计，VLESS + Reality 在当前环境下具备更强的隐蔽性与生存能力，成为一种更接近"长期可用"的方案。

## 一、连接服务器（SSH 登录）

### Windows 10 / 11 用户

按下 **Win + R** 键，输入 `cmd` 打开命令提示符，再输入以下命令（把 IP 换成你的服务器 IP）：

```bash
ssh root@你的VPS的IP
```

* 如果提示 **Are you sure you want to continue connecting?** ，输入 `yes` 并按回车。
* 当提示 **password:**  时，粘贴你的密码（注意：输入时光标不会移动，这是正常的），然后按回车键。

### macOS 用户

按 **⌘ Command + 空格键** 搜索终端（Terminal）并打开，输入与 Windows 相同的命令并输入密码登录。

---

成功连接后，你会看到一个欢迎界面，并出现 `root@xxx:~#` 这样的提示符，这意味着你已经进入了服务器。

## 二、SSH 安全配置

> ⚠️ SSH 默认端口（22）极易被爆破， **必须修改端口！** （注意：把 `2222` 修改成你自己设定的端口）

**Step 1：修改 SSH 端口**

```bash
sed -i 's/^#\?Port 22/Port 2222/' /etc/ssh/sshd_config
```

**Step 2：安装 UFW 防火墙**

（通常 VPS 服务商不会预装 UFW）

```bash
apt update && apt install ufw -y && ufw --force enable
```

**Step 3：放行 SSH 端口**

修改 SSH 端口后必须执行此命令，否则无法连接 SSH。

```bash
ufw allow 你设置的端口/tcp
```

**Step 4：放开 443 和 80 端口**

（方便 SSL 证书和加密隧道连接）

```bash
ufw allow 80/tcp && ufw allow 443/tcp
```

**Step 5：重启服务器**

```bash
reboot
```

**Step 6：重新连接（开启加密隧道）**

```bash
ssh -p 你设定的SSH端口 -L 2095:127.0.0.1:2095 root@你的服务器IP
```

如果显示如下界面，说明隧道开启成功：

## 三、安装 X-ui 面板

**开启BBR加速：**

```bash
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p
```

验证 BBR 是否开启成功

```bash
sysctl net.ipv4.tcp_congestion_control
lsmod | grep bbr
```

如果输出 `net.ipv4.tcp_congestion_control = bbr` 和 `tcp_bbr` 字样就说明成功

如果没有也没关系，现在基本上的系统都会自带BBR，如果追求网络速度，可以去询问AI如何安装就行

**使用一键安装命令：**

```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

安装过程中按以下方式回答提示：

1. 出现 `Would you like to customize the Panel Port settings?` → 输入 `y` 并回车
2. 出现 `Please set up the panel port:` → 输入 `2095`
3. 出现 `Choose an option (default 2 for IP):` → 直接回车（默认）
4. 出现 `Do you have an IPv6 address to include?` → 直接回车（跳过）
5. 出现 `Port to use for ACME HTTP-01 listener (default 80):` → 直接回车（默认）
6. 安装成功后，记录红色标记位置的信息，如图：

## 五、客户端推荐

### 1\. Windows — v2rayN

* 访问 GitHub 下载 <a href="https://github.com/2dust/v2rayN/releases" target="_blank" rel="noopener">https://github.com/2dust/v2rayN/releases</a>
* 确保你已经复制了 `vless://` 链接
* 在软件里按 `Ctrl + V` 导入
* 选中节点后按回车测试延迟，然后右键点击右下角任务栏图标 →「系统代理」→「自动配置系统代理」，即可联网

> ⚠️ **Windows 系统下使用 Tun 模式会有 DNS 泄露风险，需要禁用 Windows 智能多网卡 DNS 解析，这非常重要！**

以**管理员身份**运行 CMD，执行以下命令：

```plain
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient" /v DisableSmartNameResolution /t REG_DWORD /d 1 /f
```

之后重启 Windows 系统，再执行以下命令验证是否生效：

```plain
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient" /v DisableSmartNameResolution
```

如果提示 `DisableSmartNameResolution    REG_DWORD    0x1` 则表示生效。

完成之后可以访问这个进行测试<a href="https://browserleaks.com/dns" target="_blank" rel="noopener">https://browserleaks.com/dns</a>

如下图，如果你使用的是美国VPS，那么以下全部必须是美国，如果其中有一个是你所在的国家的图标，就说明你已经DNS泄露

![正常，DNS未泄露](https://img.alexcorner.com/image/fe6447e3-4a2e-4e6d-947f-d8344f94b4f1.webp)![DNS已泄露，这需要注意](https://img.alexcorner.com/image/1939552e-1c00-4f2f-aa58-3a944a99b71c.webp)
